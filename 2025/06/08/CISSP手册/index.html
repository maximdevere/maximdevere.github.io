CISSP 8个域详细学习重点

基于firmianay.gitbook.io网站笔记的深度解析

域1：安全与风险管理 (15%) - 核心治理域

🎯 CIA三元组 - 信息安全基石

保密性（Confidentiality）

  • 实现机制：对称密码、非对称密码
  • 数据状态：静止、传输、使用
  • 失败表现：泄漏（Disclosure）
  • 过度保护后果：导致可用性受限

完整性（Integrity）

  • 实现机制：消息摘要（散列函数）
  • 确保内容：数据未被未经授权更改
  • 失败表现：改变（Alteration）
  • 过度保护后果：可用性受限

可用性（Availability）

  • 确保授权用户任何时间都可访问
  • 失败表现：销毁（Destruction）
  • 过度提供后果：机密性和完整性损失

扩展概念

  • 真实性（Authenticity）= 完整性 + 不可否认性
  • 不可否认性（Nonrepudiation）：通过数字签名实现

🔐 AAA服务模型 - 访问控制基础

  1. 标识（Identification）：声明自己是谁
  2. 身份认证（Authentication）：验证身份真实性
  3. 授权（Authorization）：分配访问权限
  4. 审计（Auditing）：记录用户行为
  5. 可问责性（Accounting）：事后追责

🛡️ 保护机制详解

纵深防御（Defense in Depth）

  • 分层：连续使用多个控制
  • 宏观：管理、技术、物理控制组合
  • 细节：边界防火墙 + 数据中心防火墙

抽象（Abstraction）

  • 高效处理：相同属性元素统一管理
  • 典型应用：基于角色的访问控制（RBAC）

数据隐藏（Data Hiding）

  • 需要授权才能访问的数据存放方式
  • vs 隐式安全：不告知对象存在

加密（Encryption）

  • 即使获取数据也无法读取实际内容

📊 风险管理 - 量化决策基础

定量风险分析公式

风险要素三角

  • 资产（Asset）：有形价值 + 无形价值
  • 威胁（Threat）：自然 + 人为
  • 脆弱性（Vulnerability）：系统弱点

风险响应六策略

  1. 风险缓解：部署防火墙降低被攻击风险
  2. 风险转移：购买保险转移经济损失
  3. 风险接受：成本超过价值，接受风险
  4. 风险拒绝：不承认风险存在（错误做法）
  5. 风险威慑：威慑攻击者的心理战术
  6. 风险规避：彻底避免风险活动

风险变化类型

  • 固有风险：风险管理前的原始风险
  • 剩余风险：实施控制后仍存在的风险
  • 总风险：无保障措施时面临的风险
  • 控制间隙：总风险 - 剩余风险

🎭 威胁建模 - STRIDE方法论

Spoofing（欺骗）

  • 攻击方式：ARP欺骗、窃取cookie
  • 防护目标：身份验证

Tampering（篡改）

  • 攻击方式：修改数据库记录
  • 防护目标：完整性

Repudiation（否认）

  • 攻击方式：共用账号无法审计
  • 防护目标：不可否认性

Information Disclosure（信息泄露）

  • 攻击方式：图纸、薪酬、PII泄露
  • 防护目标：机密性

DoS（拒绝服务）

  • 攻击方式：DDoS攻击
  • 防护目标：可用性

Elevation of Privilege（特权提升）

  • 攻击方式：利用普通账号获取管理员权限
  • 防护目标：授权控制

📋 安全治理体系

制度层次结构

策略类型

  • 监管性策略：遵循法律要求
  • 建议性策略：定义行为和处罚
  • 信息性策略：组织知识和目标

组织角色职责

  • 高级管理层：最终责任，策略批准
  • 安全专家：具体实施
  • 资产所有者：保护责任（高级经理）
  • 托管员：执行保护任务（运维工程师）
  • 用户：系统使用者
  • 审计员：合规检查

👥 人员安全 - 全生命周期管理

招聘筛选要素

  • 工作履历验证
  • 教育背景核实
  • 同事推荐访谈
  • 犯罪记录查询
  • 药物测试检查
  • 性格评估分析
  • 社交媒体审查

在职监督机制

  • 特权蠕变（Privilege Creep）防范
  • 强制休假（1-2周）检查
  • 合谋（Collusion）预防
  • 用户实体行为分析（UEBA）

离职处理清单

  • 同时或之前禁用账户
  • 离职面谈强调NDA/NCA
  • 回收公司资产
  • 安保陪同收拾物品
  • 通知相关人员

社会工程心理学原理

  • 权威：冒充CEO下达指令
  • 恐吓：威胁解雇强迫服从
  • 共识：声称同事已同意协助
  • 稀缺性：限时优惠诱惑
  • 熟悉感：利用对熟人的信任
  • 信任：建立关系后利用
  • 紧迫性：制造时间压力

社会工程攻击类型

  • 网络钓鱼（Phishing）：通用诈骗邮件
  • 鱼叉钓鱼（Spear Phishing）：针对性攻击
  • 鲸钓（Whaling）：针对高管
  • 钓鱼短信（Smishing）：SMS诈骗
  • 钓鱼语音（Vishing）：电话诈骗
  • 商业邮件诈骗（BEC）：伪装高管邮件

📈 业务连续性计划（BCP）

制定流程

  1. 组织审查：识别所有部门和业务流程
  2. BCP团队选择：多部门代表 + 高管支持
  3. 资源需求评估：人力财力硬件软件
  4. 法律法规要求：合规底线

业务影响分析（BIA）关键指标

  • 资产价值（AV）：业务系统重要性
  • 最大允许中断时间（MTD/MTO）：业务可承受停机时间
  • 恢复时间目标（RTO）：系统恢复所需时间
  • 恢复点目标（RPO）：可接受的数据丢失量
  • 核心要求：RTO < MTD

连续性规划保护资产

  1. 人员：最高优先级，人命关天
  2. 建筑设施：加固预备 + 替代站点
  3. 基础设施：物理加固 + 替代系统

⚖️ 法律法规合规

知识产权保护

  • 版权：文学音乐影视代码，死后70年
  • 商标：公司标识口号，10年续期无限
  • 专利：发明创新，20年保护期
  • 商业秘密：核心竞争力，签署NDA保护

重要法规详解

GDPR核心条款

  • 合法公平透明：遵循法律，公开处理活动
  • 目的限制：文档化声明收集目的
  • 数据最小化：仅收集必需数据
  • 准确性：确保数据正确
  • 存储限制：遵守被遗忘权
  • 可问责性：对数据使用负责

美国隐私法体系

  • HIPAA：医疗数据保护，要求BAA协议
  • COPPA：13岁以下儿童需家长同意
  • GLBA：金融机构客户信息保护
  • FERPA：教育机构学生隐私权

行业合规标准

  • PCI DSS：信用卡数据安全
  • SOX：财务数据完整性
  • FISMA：联邦政府网络安全

域2：资产安全 (10%) - 数据保护核心

📁 数据分类体系

政府分类标准

  • 绝密（Top Secret）：国家安全重大危害
  • 机密（Secret）：国家安全严重危害
  • 受限（Confidential）：国家安全一般危害
  • 非密（Unclassified）：无害信息

商业分类标准

  • 机密（Confidential）：竞争敏感信息
  • 私有（Private）：内部使用信息
  • 敏感（Sensitive）：需要保护但可有限共享
  • 公开（Public）：可公开发布信息

🔄 数据生命周期管理

六阶段模型

  1. 创建（Creation）：数据产生，初始分类
  2. 存储（Storage）：安全存储，访问控制
  3. 使用（Use）：授权访问，活动监控
  4. 共享（Sharing）：传输保护，接收方验证
  5. 归档（Archiving）：长期保存，合规要求
  6. 销毁（Destruction）：安全清除，证书保存

每阶段安全控制

  • 数据创建：自动分类标记
  • 数据存储：加密、备份、访问控制
  • 数据使用：DLP、水印、活动审计
  • 数据共享：传输加密、数字签名
  • 数据归档：完整性验证、定期检查
  • 数据销毁：多次覆写、物理销毁

🎭 数据角色与职责

数据所有者（Data Owner）

  • 通常是高级经理
  • 负责数据保护最终责任
  • 决定数据分类级别
  • 授权访问权限

数据管理员（Data Custodian）

  • 执行日常保护任务
  • 实施技术控制措施
  • 监控数据访问活动
  • 执行备份和恢复

数据处理者（Data Processor）

  • 代表数据控制者处理数据
  • 必须按指令处理数据
  • 需要签订处理协议
  • 承担有限责任

数据主体（Data Subject）

  • 数据所指向的个人
  • 拥有数据权利
  • 可要求访问、纠正、删除

🔒 隐私保护技术

去标识化技术

  • 匿名化（Anonymization）：不可逆去除标识
  • 假名化（Pseudonymization）：可逆的标识替换
  • K-匿名：至少K个记录具有相同属性
  • 差分隐私：添加数学噪声保护

隐私增强技术（PET）

  • 同态加密：加密状态下计算
  • 安全多方计算：分布式隐私计算
  • 零知识证明：证明而不泄露信息
  • 联邦学习：分布式机器学习

📊 数据质量管理

数据质量维度

  • 准确性：数据反映真实情况
  • 完整性：数据完整无缺失
  • 一致性：不同来源数据一致
  • 及时性：数据反映当前状态
  • 有效性：数据符合业务规则

数据治理流程

  • 数据资产清单：识别所有数据资产
  • 数据血缘分析：追踪数据来源和流向
  • 数据质量监控：持续监测数据质量
  • 元数据管理：管理数据的数据

域3：安全架构和工程 (13%) - 技术基础域

🔐 密码学深度解析

信息安全四大支柱实现

  • 保密性：对称加密（AES）+ 非对称加密（RSA）
  • 完整性：散列函数（SHA-256）+ HMAC
  • 身份验证：零知识证明 + 数字证书
  • 不可否认性：数字签名（私钥签名，公钥验证）

密码学基本原理

  • 显式安全（Kerckhoffs原则）：算法公开仍安全
  • 隐式安全：算法保密才安全（错误理念）
  • 科学分类：
    • 密码术（Cryptography）：创建密码
    • 密码分析（Cryptanalysis）：破解密码
    • 密码学（Cryptology）：两者结合

布尔数学与逻辑运算

密码学关键概念

  • 模函数（mod）：除法求余运算
  • 单向函数：A→B容易，B→A困难
  • Nonce：一次性随机数（如IV）
  • 零知识证明：证明知道秘密而不泄露秘密
  • 分隔知识：M of N控制，需要M个人同时在场

🔑 对称加密算法详解

算法分类

  • 流密码：逐字符/位加密（RC4、一次性密码本）
  • 块密码：固定长度块加密（DES、AES）

主要算法比较

分组密码工作模式

  • ECB（电子密码本）：每块独立加密，不安全
  • CBC（密码块链接）：前一块密文参与下一块加密
  • CFB（密码反馈）：流密码版本的CBC
  • OFB（输出反馈）：生成密钥流加密
  • CTR（计数器）：计数器生成密钥流
  • GCM（伽罗瓦计数器）：CTR+认证，AEAD模式

对称加密缺点

  • 密钥分发问题：如何安全交换密钥
  • 不提供不可否认性：无法确认发送方
  • 缺乏可伸缩性：n个用户需要n(n-1)/2个密钥
  • 密钥生命周期短：人员变动影响全局

🗝️ 非对称加密算法

主要算法特点

  • RSA：基于大整数分解，专利已过期
  • ECC：椭圆曲线，160位ECC ≈ 1024位RSA
  • El Gamal：基于Diffie-Hellman，密文长度翻倍
  • Diffie-Hellman：仅用于密钥交换

非对称加密优势

  • 密钥分发简单：公钥公开，私钥自己保管
  • 提供不可否认性：私钥签名唯一性
  • 可伸缩性好：n个用户仅需2n个密钥
  • 密钥生命周期长：新用户加入不影响他人

量子计算威胁

  • 量子优势：解决经典计算难题
  • 威胁算法：RSA、DH等基于数学难题的算法
  • 应对策略：后量子密码学算法研究
  • 时间考虑：评估数据敏感期vs量子计算普及时间

#️⃣ 散列函数与完整性

散列函数五大要求

  1. 输入任意长度
  2. 输出固定长度
  3. 计算相对容易
  4. 单向不可逆
  5. 无碰撞性（不同输入不产生相同输出）

主要散列算法

消息认证码（MAC）

  • 目的：防止传输中被篡改
  • HMAC：密钥+消息一起计算散列
  • 提供：完整性+认证，不提供不可否认性

✍️ 数字签名技术

数字签名过程

  1. 发送方：消息→散列→私钥加密→数字签名
  2. 接收方：公钥解密→对比消息散列→验证真实性

数字签名算法

  • RSA：可用于加密和签名
  • DSA：专门用于数字签名
  • ECDSA：椭圆曲线数字签名

数字证书（X.509标准）

  • 版本号、序列号、签名算法标识
  • 颁发者、有效期、主体名称
  • 主体公钥、扩展字段

PKI基础设施

  • CA（证书颁发机构）：签发和管理证书
  • RA（注册机构）：验证用户身份
  • CRL（证书撤销列表）：已撤销证书清单
  • OCSP：在线证书状态协议

🏗️ 安全架构原则

核心设计原则

  • 最小权限：仅分配必需的最小权限
  • 职责分离：关键操作需多人参与
  • 深度防御：多层安全控制
  • 故障安全：系统故障时保持安全状态

故障处理模式

  • 故障弱化（Fail-soft）：部分功能继续运行
  • 故障安全（Fail-secure）：物理安全优先，门禁关闭
  • 故障安全（Fail-safe）：生命安全优先，门禁开启
  • 故障开放（Fail-open）：可用性优先，防火墙bypass
  • 故障关闭（Fail-closed）：安全优先，阻断所有流量

安全模型详解

Bell-LaPadula模型（机密性）

  • 简单安全属性：不准向上读（No Read Up）
  • *安全属性：不准向下写（No Write Down）
  • 自由安全属性：自主访问控制矩阵

Biba模型（完整性）

  • 简单完整性属性：不准向下读
  • *完整性属性：不准向上写
  • 与Bell-LaPadula正好相反

Clark-Wilson模型（完整性）

  • CDI（受约束数据项）：需要保护的数据
  • UDI（无约束数据项）：不需要保护的数据
  • IVP（完整性验证过程）：验证数据完整性
  • TP（转换过程）：唯一可修改CDI的过程

Chinese Wall模型（动态权限）

  • 防止利益冲突
  • 根据用户历史访问动态调整权限
  • 如访问公司A数据后，不能访问竞争对手公司B数据

💻 系统架构安全

处理器保护机制

  • 保护环（Protection Rings）：
    • Ring 0：内核，最高权限
    • Ring 1：操作系统其他部分
    • Ring 2：I/O驱动和系统应用
    • Ring 3：用户应用程序

进程状态管理

  • 就绪状态（Ready）：等待CPU分配
  • 等待状态（Waiting）：等待资源或事件
  • 运行状态（Running）：正在执行
    • 问题状态（Problem）：用户模式
    • 监管状态（Supervisory）：特权模式
  • 停止状态（Stopped）：进程终止

内存管理

  • 实际内存（Real Memory）：物理RAM
  • 虚拟内存（Virtual Memory）：硬盘模拟内存
  • 高速缓存（Cache）：CPU和内存间缓冲
  • 寄存器（Registers）：CPU板载存储

内存寻址方式

  • 寄存器寻址：直接使用CPU寄存器
  • 立即寻址：数据包含在指令中
  • 直接寻址：指令包含实际地址
  • 间接寻址：通过另一个地址访问
  • 基址+偏移：基地址+偏移量计算

🖥️ 虚拟化与云计算

虚拟化类型

  • Type 1 Hypervisor：直接安装在硬件（ESXi）
  • Type 2 Hypervisor：安装在操作系统上（VMware）

虚拟化安全挑战

  • VM逃逸：虚拟机突破隔离攻击宿主机
  • VM蔓延：大量未管理虚拟机
  • 影子IT：未经授权的IT部署
  • 虚拟机监控程序攻击

容器安全

  • 容器vs虚拟机：共享内核vs独立操作系统
  • 容器镜像安全：基础镜像漏洞扫描
  • 运行时保护：容器行为监控
  • 编排安全：Kubernetes安全配置

云服务模型安全

  • IaaS：基础设施即服务，客户负责操作系统以上

🔐 网络安全协议

SSL/TLS详解

  • SSL：安全套接字层，已弃用
  • TLS：传输层安全，当前标准
  • 握手过程：
    1. 客户端发送支持的加密套件
    2. 服务器选择加密套件并发送证书
    3. 客户端验证证书并生成预主密钥
    4. 双方计算会话密钥开始加密通信

IPSec架构

  • AH（认证头）：提供完整性和认证，不加密
  • ESP（封装安全载荷）：提供机密性、完整性、认证
  • 传输模式：仅加密载荷
  • 隧道模式：加密整个IP包
  • IKE：密钥交换协议

VPN类型

  • 站点到站点VPN：连接不同地理位置网络
  • 远程访问VPN：个人设备接入企业网络
  • 分离隧道：部分流量走VPN，部分直接访问互联网（安全风险）

📡 无线网络安全深度

Wi-Fi安全标准演进

无线攻击方法

  • War Driving：开车寻找无线网络
  • War Chalking：标记发现的无线网络
  • 恶意接入点：伪造合法AP
  • Evil Twin：克隆合法AP欺骗用户
  • Deauth攻击：强制断开客户端连接
  • WPS攻击：利用WPS PIN漏洞
  • Krack攻击：WPA2密钥重装攻击

无线安全最佳实践

  • 更改默认SSID和管理密码
  • 禁用WPS功能
  • 使用强加密（WPA3/WPA2）
  • 隐藏SSID（安全性有限）
  • MAC地址过滤（可被绕过）
  • 定期更新固件
  • 使用企业级认证（802.1X）

域5：身份与访问管理 (13%) - 访问控制核心

🔑 身份管理基础

身份验证三要素

  • 知识因素（Something you know）：
    • 密码、PIN码、安全问题
    • 易被窃取、分享、忘记
  • 持有因素（Something you have）：
    • 硬件令牌、智能卡、手机
    • 可能丢失、被盗、损坏
  • 生物因素（Something you are）：
    • 指纹、虹膜、面部、声纹
    • 难以伪造，但有错误率

多因素认证（MFA）

  • 双因素：两种不同类型要素
  • 多因素：三种或更多要素
  • 安全性：组合使用比单一要素更安全
  • 用户体验：平衡安全与便利性

生物识别技术

  • 错误率指标：
    • FAR（错误接受率）：错误接受非授权用户
    • FRR（错误拒绝率）：错误拒绝授权用户
    • CER（交叉错误率）：FAR=FRR的点，越低越好
  • 生物识别类型：
    • 生理特征：指纹、虹膜、面部、手掌
    • 行为特征：签名、击键、步态、声音

🚪 访问控制模型详解

自主访问控制（DAC）

  • 资源所有者决定访问权限
  • 权限可以传递和委派
  • 灵活但难以集中管理
  • 容易权限蔓延
  • 典型实现：Windows NTFS权限

强制访问控制（MAC）

  • 系统强制执行安全策略
  • 基于安全标签和级别
  • 用户无法修改访问权限
  • 适用于高安全环境
  • 典型实现：军方分级系统

基于角色的访问控制（RBAC）

  • 基于用户角色分配权限
  • 角色继承和层次结构
  • 职责分离原则
  • 简化权限管理
  • 典型实现：企业ERP系统

基于属性的访问控制（ABAC）

  • 基于多种属性动态授权
  • 属性包括：主体、客体、环境、动作
  • 最灵活的访问控制模型
  • 复杂的策略语言（XACML）
  • 适用于云环境和复杂场景

🎫 身份联邦与单点登录

SAML（安全断言标记语言）

  • 基于XML的身份联邦标准
  • 组件：
    • 身份提供者（IdP）：认证用户身份
    • 服务提供者（SP）：提供业务服务
    • 断言：身份和属性声明
  • 流程：用户→IdP认证→SAML断言→SP授权访问

OAuth 2.0授权框架

  • 角色：
    • 资源所有者：用户
    • 客户端：第三方应用
    • 授权服务器：颁发访问令牌
    • 资源服务器：受保护资源
  • 授权类型：
    • 授权码：最安全，适用于Web应用
    • 隐式：适用于移动应用
    • 客户端凭据：应用间通信
    • 密码：仅可信应用

OpenID Connect

  • OAuth 2.0之上的身份层
  • 提供身份验证功能
  • ID令牌：包含用户身份信息
  • 用户信息端点：获取用户详细信息

🔐 密码安全与管理

密码策略设计

  • 长度要求：最少8-12位
  • 复杂度：大小写、数字、特殊字符
  • 历史记录：避免重复使用

密码攻击与防护

  • 字典攻击：常用密码列表
  • 暴力攻击：穷举所有可能
  • 彩虹表：预计算哈希值
  • 防护措施：
    • 加盐（Salt）：随机数据增加复杂度
    • 密钥延展：多次哈希计算
    • 账户锁定：限制尝试次数

密码管理器

  • 生成复杂随机密码

  • 自动填充功能
  • 主密码保护
  • 多设备同步

  • 目的：独立验证合规性
  • 特点：

💻 软件安全测试

静态应用安全测试（SAST）

  • 源代码分析
  • 不需要运行程序
  • 早期发现漏洞
  • 高覆盖率，误报率高
  • 工具：SonarQube、Checkmarx、Veracode

动态应用安全测试（DAST）

  • 黑盒运行时测试
  • 模拟真实攻击
  • 低误报率，覆盖率有限
  • 工具：OWASP ZAP、Burp Suite、AppScan

交互式应用安全测试（IAST）

  • 结合SAST和DAST优势
  • 运行时代码分析
  • 精确漏洞定位
  • 低误报率，高覆盖率

软件组合分析（SCA）

  • 开源组件漏洞扫描
  • 许可证合规检查
  • 组件版本管理
  • 供应链安全

    5. 失效的功能级授权
    6. 批量分配

🛡️ 应用程序安全威胁

OWASP Top 10 2021详解

1. 失效的访问控制

  • 垂直权限提升：普通用户获得管理员权限
  • 水平权限提升：访问其他用户的数据
  • IDOR（不安全直接对象引用）
  • 防护措施：
    • 默认拒绝访问
    • 最小权限原则
    • 访问控制矩阵
    • 服务端验证

2. 加密机制失效

  • 明文传输敏感数据
  • 使用弱加密算法
  • 密钥管理不当
  • 防护措施：
    • 强制HTTPS传输
    • 使用强加密算法
    • 安全密钥管理
    • 敏感数据分类

3. 注入攻击

  • SQL注入：恶意SQL代码注入
  • NoSQL注入：非关系型数据库注入
  • LDAP注入：目录服务注入
  • 命令注入：操作系统命令注入
  • 防护措施：
    • 参数化查询
    • 输入验证
    • 最小权限数据库账户
    • WAF防护

4. 不安全设计

  • 缺乏安全设计原则
  • 威胁建模不足
  • 安全控制缺失
  • 防护措施：
    • 安全设计原则
    • 威胁建模实践
    • 安全架构审查
    • 防御性编程

5. 安全配置错误

  • 默认账户和密码
  • 不必要的功能启用
  • 错误页面信息泄露
  • 防护措施：
    • 安全基线配置
    • 定期配置审查
    • 自动化配置管理
    • 最小化安装

6. 易受攻击和过时的组件

  • 使用已知漏洞组件
  • 不及时更新补丁
  • 使用不受支持的版本
  • 防护措施：
    • 组件清单管理
    • 漏洞扫描
    • 及时更新补丁
    • 依赖管理工具

7. 身份验证和会话管理缺陷

  • 弱密码策略
  • 会话固定攻击
  • 会话劫持风险
  • 防护措施：
    • 强身份验证
    • 安全会话管理
    • 多因素认证
    • 会话超时控制

8. 软件和数据完整性故障

  • 不安全的反序列化
  • 供应链攻击
  • 自动更新机制缺陷
  • 防护措施：
    • 数字签名验证
    • 安全反序列化
    • 供应链安全
    • 完整性检查

9. 安全日志记录和监控故障

  • 重要事件未记录
  • 日志信息不足
  • 监控告警缺失
  • 防护措施：
    • 全面日志记录
    • 实时监控告警
    • 日志保护措施
    • 事件响应集成

10. 服务器端请求伪造（SSRF）

  • 内网服务访问
  • 云元数据泄露
  • 端口扫描攻击
  • 防护措施：
    • URL白名单验证
    • 网络分段
    • 输入验证
    • 防火墙规则

🔍 安全代码审查

静态代码分析（SAST）

  • 优势：
    • 早期漏洞发现
    • 完整代码覆盖
    • 开发阶段集成
    • 修复成本较低
  • 局限性：
    • 误报率较高
    • 无法检测逻辑漏洞
    • 需要源代码访问
    • 运行时行为未知
  • 最佳实践：
    • 制定编码标准
    • 集成开发环境
    • 定期规则更新
    • 误报管理机制

手工代码审查

  • 审查重点：
    • 输入验证机制
    • 输出编码实现
    • 身份验证逻辑
    • 授权检查机制
    • 错误处理方式
    • 日志记录内容
  • 审查流程：
    • 制定审查清单
    • 分配审查角色
    • 集中讨论问题
    • 跟踪修复进度
    • 验证修复效果

🌐 Web应用安全

跨站脚本攻击（XSS）

  • 反射型XSS：恶意脚本通过URL参数注入
  • 存储型XSS：恶意脚本存储在服务器上
  • DOM型XSS：客户端脚本漏洞
  • 防护措施：
    • 输出编码
    • 内容安全策略（CSP）
    • HTTPOnly Cookie
    • 输入验证

跨站请求伪造（CSRF）

  • 攻击原理：利用用户认证状态执行恶意操作
  • 防护措施：
    • CSRF令牌验证
    • SameSite Cookie属性
    • 双重提交Cookie
    • 验证Referer头

会话管理安全

  • 会话ID安全：
    • 随机性强
    • 长度足够
    • 定期轮换
    • 安全传输
  • 会话保护：
    • HTTPOnly属性
    • Secure属性
    • SameSite属性
    • 会话超时
