Help HTB 渗透测试笔记

整体渗透思路

  1. 端口扫描发现22、80、3000端口，80端口重定向到help.htb
  2. 80端口发现HelpDeskZ帮助台系统，3000端口发现GraphQL API
  3. GraphQL schema introspection获取用户凭据（用户名和MD5密码哈希）
  4. 使用hashcat破解MD5哈希获得明文密码
  5. 利用HelpDeskZ文件上传漏洞（CVE-2016-10134）上传PHP webshell
  6. 通过时间戳暴力破解找到上传文件位置，获得webshell
  7. 反向shell获得help用户权限
  8. 利用内核漏洞CVE-2017-16995提权至root

端口扫描

全端口扫描

参数解析：

  • -sT: TCP连接扫描，完整三次握手
  • --min-rate 10000: 最小发包速率，加快扫描速度
  • -p-: 扫描全部65535端口

详细服务扫描

参数解析：

  • -sC: 使用默认NSE脚本集合
  • -sV: 版本探测识别服务版本
  • -T3: 时序模板，正常速度扫描

扫描结果：

  • 22/tcp: SSH (OpenSSH 7.2p2)
  • 80/tcp: HTTP (Apache 2.4.18) - 重定向到help.htb
  • 3000/tcp: HTTP (Node.js Express)

域名配置和Web服务分析

域名解析配置

原因： HTTP服务重定向到help.htb域名，需要配置本地解析

HTTP服务分析（80端口）

访问http://help.htb发现HelpDeskZ帮助台系统

目录枚举

参数解析：

  • dir: 目录暴力破解模式
  • -u: 目标URL
  • -w: 字典文件路径

重要发现：

  • /support/ - HelpDeskZ主目录
  • /support/uploads/tickets/ - 文件上传目录

GraphQL API分析（3000端口）

Schema Introspection查询

GraphQL Introspection原理：

  • __schema: GraphQL内省查询，获取API结构信息
  • queryType: 查询根类型的字段信息
  • fields: 获取所有可用字段和类型

关键发现：

  • 存在user字段，返回类型为User
  • User类型包含username和password字段

敏感数据获取

GraphQL查询语法：

  • { user { username password } }: 查询user对象的username和password字段
  • GraphQL允许精确指定需要的字段，减少数据传输

获得凭据：

  • 用户名: helpme@helpme.com
  • 密码哈希: 5d3c93182bb20f07b994a7f617e99cff (MD5格式)

密码破解

MD5哈希识别

Hashcat暴力破解

参数解析：

  • -m 0: 指定哈希类型为MD5
  • --force: 强制执行（忽略警告）
  • /usr/share/wordlists/rockyou.txt: 常用密码字典

破解结果： godhelpmeplz

HelpDeskZ文件上传漏洞利用

漏洞分析（CVE-2016-10134）

漏洞机制：

  1. HelpDeskZ允许上传文件作为工单附件
  2. 上传文件被重命名为：md5(原文件名 + 当前时间戳).扩展名
  3. 文件保存在/support/uploads/tickets/目录
  4. 攻击者可通过时间戳暴力破解找到上传文件

登录HelpDeskZ

使用获得的凭据：

  • 用户名: helpme@helpme.com

URL编码解析：

系统信息收集

内核漏洞识别

目标系统： Ubuntu 16.04.4 LTS，内核版本4.4.0-116-generic 已知漏洞： CVE-2017-16995（eBPF verifier漏洞）

漏洞利用

CVE-2017-16995漏洞原理：

  • eBPF（extended Berkeley Packet Filter）verifier存在整数溢出漏洞
  • 攻击者可构造特殊的eBPF程序绕过验证
  • 在内核空间执行任意代码，实现权限提升
