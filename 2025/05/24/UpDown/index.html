UpDown HTB 渗透测试笔记

整体渗透思路

  1. 端口扫描发现80和22端口，Web应用”Is my Website up ?”
  2. 目录扫描发现/dev目录，子域名枚举找到dev.siteisup.htb
  3. 发现.git仓库暴露，使用git-dumper获取完整源代码
  4. 源码审计发现多个漏洞：访问控制绕过、LFI、文件上传、SSRF
  5. 绕过.htaccess访问控制（Special-Dev头部），利用文件上传+竞态条件获得RCE
  6. 发现SUID程序/home/developer/dev/siteisup，利用Python代码注入提权至developer
  7. 发现sudo权限easy_install，使用GTFOBins技术提权至root

端口扫描

快速端口发现

参数解析：

  • -sT: TCP连接扫描，完整三次握手，更准确但较慢
  • --min-rate 10000: 最小发包速率10000包/秒，加快扫描速度
  • -p-: 扫描全部65535个端口

详细服务识别

参数解析：

  • -sC: 使用默认NSE脚本集合，等同于 --script=default
  • -sV: 版本探测，识别服务版本信息

扫描结果：

  • 22/tcp: SSH OpenSSH 8.2p1 Ubuntu
  • 80/tcp: HTTP Apache 2.4.41 (Ubuntu)

信息收集

域名发现

访问80端口发现网站标题”Is my Website up ?”，页面源码中发现域名siteisup.htb

目录暴力破解

参数解析：

  • -w: 指定字典文件
  • -t 100: 并发线程数100
  • -mc 200,302,301: 匹配HTTP状态码
  • -u http://siteisup.htb/FUZZ: 目标URL，FUZZ为占位符

发现： /dev 目录 (Status: 301重定向)

子域名枚举

参数解析：

  • -c: 彩色输出
  • -H "Host: FUZZ.siteisup.htb": 设置Host头进行虚拟主机枚举
  • --hc 400: 隐藏HTTP 400状态码响应
  • 技术原理： 通过修改Host头部测试不同子域名，服务器根据Host头路由到不同虚拟主机

发现子域名： dev.siteisup.htb

Git仓库发现

参数解析：

  • dir: 目录暴力破解模式
  • -u: 目标URL
  • -w: 字典文件

关键发现： /.git/HEAD (Status: 200) - Git仓库暴露

Git仓库利用

Git仓库转储

工具原理：

  • git-dumper通过访问.git目录的各种文件重建完整仓库
  • 利用Git的内部文件结构（objects、refs、index等）恢复源代码
  • 比直接访问.git/config更全面，可获取完整提交历史

源码文件结构

源代码分析

.htaccess访问控制分析

机制解析：

  • SetEnvIfNoCase: 不区分大小写设置环境变量
  • 当HTTP头Special-Dev值为only4dev时，设置环境变量Required-Header
  • Order Deny,Allow: 先拒绝所有，再根据Allow规则允许
  • Allow from env=Required-Header: 只允许有该环境变量的请求

绕过方法： 添加HTTP头部 Special-Dev: only4dev

index.php路由分析

漏洞分析：

  • LFI漏洞： 直接包含用户输入的文件
  • 过滤绕过： 正则只过滤特定目录名，可用../绕过
  • 文件扩展名： 自动添加.php后缀，限制了文件类型

checker.php文件上传分析

漏洞点：

  • 扩展名绕过： .phar扩展名未被过滤，但PHP仍可执行
  • 竞态条件： 文件上传后到删除前存在时间窗口
  • 目录可预测： 使用md5(time())生成目录名，时间可预测

漏洞利用

竞态条件+文件上传RCE

1. 构造恶意.phar文件

技术说明：

  • 文件伪装： 前两行是正常URL，绕过内容检测
  • 反向Shell： 使用proc_open执行bash反向连接
  • 重定向机制： >& 将标准输出和错误都重定向到网络连接

2. 启动监听器

参数解析：

  • -n: 不进行DNS解析，提高连接速度
  • -l: 监听模式
  • -v: 详细输出
  • -p 443: 监听443端口

3. 利用时序窗口

攻击流程：

  1. 上传shell.phar到 http://dev.siteisup.htb/?page=checker
  2. 添加绕过头部: Special-Dev: only4dev
  3. 利用checker.php处理URL的延时（httpbin.org/delay/5）
  4. 在文件被unlink删除前访问执行

时间窗口原理：

  • checker.php先保存文件，再处理URL内容
  • 处理延时URL时，文件仍存在服务器上
  • 此时通过LFI包含上传的文件即可执行PHP代码

4. 获得初始Shell

权限提升1: www-data → developer

系统信息收集

SUID程序发现

关键发现：

SUID权限分析：

  • s位表示SUID权限，执行时以文件所有者权限运行
  • 文件所有者是developer，www-data组可执行
  • 执行时将获得developer用户权限

程序逆向分析

关键字符串发现：

程序工作原理：

  • C程序调用Python脚本处理用户输入
  • 用户输入传递给Python脚本的URL验证函数
  • Python的input()函数可被代码注入

Python代码注入利用

注入原理：

  • Python的input()函数在某些情况下可执行Python代码
  • __import__('os').system('bash')动态导入os模块并执行系统命令
  • 由于SUID权限，bash以developer权限启动

成功提权： developer@updown:/home/developer/dev$

获取稳定SSH访问

保存并使用私钥：

权限提升2: developer → root

Sudo权限检查

输出分析：

  • (ALL): 可以以任何用户身份执行
  • NOPASSWD: 无需输入密码
  • /usr/local/bin/easy_install: 可执行的具体程序

GTFOBins技术利用

GTFOBins原理：

  • GTFOBins是Unix二进制程序的权限提升技术集合
  • 利用程序的设计功能实现意外的权限提升
  • easy_install可以执行setup.py中的任意Python代码

easy_install提权利用

技术解析：

  • mktemp -d: 创建临时目录
  • os.execl(): 替换当前进程执行新程序
  • sh <$(tty) >$(tty) 2>$(tty): 将shell的输入输出重定向到当前终端
  • easy_install执行setup.py时会以root权限运行Python代码

成功获得root： root@updown:/tmp/tmp.6bPxETrQln#
