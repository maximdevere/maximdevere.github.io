Sau HTB 渗透测试笔记

整体渗透思路

  1. 端口扫描发现55555端口运行Request-Baskets v1.2.1
  2. 搜索CVE发现CVE-2023-27163 SSRF漏洞
  3. 利用SSRF访问内网80端口，发现Maltrail v0.53应用
  4. 利用Maltrail登录功能RCE漏洞获得puma用户shell
  5. 发现sudo systemctl权限，利用pager逃逸技术提权至root

端口扫描

初始扫描

参数解析：

  • -Pn: 跳过主机发现，直接进行端口扫描（假设主机存活）
  • 适用于禁ping的主机或防火墙过滤ICMP的环境

扫描结果：

  • 55555/tcp: HTTP服务（非标准端口）

详细服务识别

参数解析：

  • -sC: 默认脚本扫描
  • -sV: 版本识别
  • -p 55555: 指定端口扫描

Web服务分析

应用识别

访问 http://10.129.229.26:55555/web 发现：

Request-Baskets功能分析：

  • HTTP请求收集服务
  • 允许创建”basket”收集和转发HTTP请求
  • 提供请求转发和代理功能

CVE漏洞搜索

搜索技巧：

  • 软件名 + 版本号 + CVE 是快速发现已知漏洞的有效方法
  • 关注GitHub Issues、安全公告、漏洞数据库

发现漏洞：

  • CVE-2023-27163: Server-Side Request Forgery (SSRF)
  • 影响版本：request-baskets ≤ v1.2.1
  • 漏洞组件：/api/baskets/{name}

SSRF漏洞利用

漏洞原理

Request-baskets的basket配置功能存在SSRF漏洞：

关键参数解析：

  • forward_url: 转发目标URL（可控制访问任意内网地址）
  • proxy_response: 启用时返回目标服务器的响应内容
  • expand_path: 保持请求路径

获取现成Exploit

内网服务探测

探测原理：

  • 创建basket并配置转发到内网地址
  • 通过代理响应功能获取内网服务信息

内网应用发现

响应分析：

发现内网应用： Maltrail v0.53 网络威胁检测系统

内网应用攻击

目录枚举

参数解析：

  • --exclude-length 92: 排除响应长度为92的结果（通常是404页面）
  • -b 502: 排除502状态码（Bad Gateway）

发现端点：

  • /login (Status: 401) - 认证端点
  • /counts (Status: 401) - 统计端点
  • /events (Status: 401) - 事件端点

Maltrail RCE漏洞

漏洞研究：

  • Maltrail v0.53存在登录功能命令注入漏洞
  • 漏洞位置：/login POST请求的username参数
  • 无需认证即可触发

漏洞原理：

反弹Shell利用

监听器参数：

  • -l: 监听模式
  • -n: 不进行DNS解析
  • -v: 详细输出
  • -p 4444: 监听端口4444

Payload解析：

  • ;bash -c '...': 分号结束前一个命令，执行新命令
  • bash -i: 交互式bash shell
  • >& /dev/tcp/IP/PORT: 重定向输入输出到TCP连接
  • 0>&1: 标准输入重定向到标准输出

成功获得shell： puma@sau

权限提升

Shell稳定化

稳定化技术：

  • pty.spawn(): 生成伪终端，支持Tab补全等功能
  • export TERM=xterm: 设置终端类型，支持颜色显示

权限枚举

sudo权限发现：

权限分析：

  • (ALL : ALL): 可以以任何用户和组身份执行
  • NOPASSWD: 无需输入密码
  • 限制命令：只能执行systemctl status trail.service

Pager逃逸技术

漏洞原理：

  1. systemctl status 使用pager（通常是less）显示长输出
  2. Pager程序继承了systemctl的执行权限（root）
  3. Less pager支持!command语法执行shell命令
  4. 通过pager执行的命令将以root权限运行

利用步骤：

在less pager界面中：

技术细节：

  • Less pager的!功能允许执行外部命令
  • 由于less以root权限运行，执行的命令也具有root权限
  • 这是一种经典的pager逃逸技术

成功获得root权限： root@sau

关键技术总结

SSRF攻击技术：

  • Request-baskets配置转发URL
  • 利用proxy_response获取内网响应
  • 内网服务发现和端口扫描

命令注入技术：

  • POST参数注入

  • Sudo权限枚举
