Searcher HTB 渗透测试笔记

整体渗透思路

这是一个典型的Web应用渗透测试案例，攻击路径为：端口扫描 → Web应用枚举 → CVE漏洞利用获得初始访问 → 信息收集发现凭据 → 横向移动获得稳定shell → sudo权限滥用实现权限提升。

关键点在于：

  1. 彻底的信息收集和枚举
  2. 对已知CVE的研究和利用
  3. 在应用文件中挖掘敏感信息（Git配置）
  4. 识别和利用系统配置错误（sudo权限）

目标信息

  • 目标IP: 10.129.2.69
  • 难度: Easy
  • 系统: Linux

1. 信息收集 (Reconnaissance)

端口扫描

参数解释:

  • -sT: TCP连接扫描，完成三次握手，更准确但较慢
  • --min-rate 10000: 最小发包速率10000/秒，加快扫描速度
  • -p-: 扫描所有65535个端口
  • 使用原因: 快速发现所有开放端口，避免遗漏服务

参数解释:

  • -sC: 使用默认NSE脚本，获取更多服务信息
  • -sV: 版本检测，识别服务版本
  • -T3: 时间模板3（正常速度），平衡速度和准确性
  • -p 22,80,8080: 只扫描发现的开放端口
  • 使用原因: 获取详细的服务版本信息，为后续漏洞研究做准备

扫描结果:

  • 22/tcp: SSH OpenSSH 8.9p1 Ubuntu
  • 80/tcp: Apache httpd 2.4.52 (重定向到 searcher.htb)
  • 8080/tcp: 关闭

域名配置

参数解释:

  • tee -a: 追加内容到文件末尾，同时输出到标准输出
  • 使用原因: Web服务重定向到域名，需要本地DNS解析才能访问

2. Web应用分析

网站访问

使用原因: 命令行快速查看网页内容和响应头信息

网站信息发现

  • URL: http://searcher.htb
  • 技术栈: Flask + Searchor 2.4.0
  • 功能: 搜索引擎聚合器

目录枚举

参数解释:

  • dir: 目录爆破模式
  • -u: 目标URL
  • -w: 指定字典文件
  • 使用原因: 发现隐藏的目录和文件，可能包含敏感信息或管理界面

3. 漏洞发现与利用

CVE研究

从网页发现关键信息：Powered by Flask and Searchor 2.4.0

使用原因: Searchor 2.4.0是较老版本，可能存在已知安全漏洞

发现: Searchor 2.4.0 存在命令注入漏洞

  • exploit: https://github.com/nikn0laty/Exploit-for-Searchor-2.4.0-Arbitrary-CMD-Injection

下载和准备exploit

参数解释:

  • raw/main: GitHub的原始文件路径，获取纯文本内容而非HTML页面
  • 使用原因: 需要下载可执行脚本，raw路径确保获得正确的文件内容

使用原因: 下载的脚本默认没有执行权限，需要手动添加

获取反向Shell

参数解释:

  • -l: 监听模式
  • -v: 详细输出
  • -n: 不进行DNS解析，加快连接速度
  • -p 9001: 监听9001端口
  • 使用原因: 建立监听器接收目标机器的反向连接

参数解释:

  • searcher.htb: 目标主机
  • 10.10.16.13: 攻击者IP（VPN接口IP）
  • 使用原因: 利用命令注入漏洞获得目标系统的shell访问

成功获得反向shell: svc@busqueda:/var/www/app

4. 后渗透 - 用户权限

Shell稳定化

使用原因: 反向shell功能有限，稳定化后可以使用tab补全、历史记录等功能

获取用户Flag

结果: f79ac2816d2a69998bfa6abf7a5d4d8e

系统信息收集

使用原因: 了解当前权限级别和所属组

使用原因: 了解操作系统版本，为后续权限提升选择合适的方法

参数解释:

  • grep -E: 使用扩展正则表达式
  • /bin/(bash|sh): 匹配有shell权限的用户
  • 使用原因: 识别系统中的其他用户，为横向移动做准备

5. 信息挖掘与横向移动

Git仓库发现

参数解释:

  • -name ".git": 查找名为.git的目录
  • -type d: 只查找目录
  • 使用原因: Git仓库经常包含敏感信息，如配置文件、历史记录中的凭据

Git配置分析

使用原因: Git配置文件可能包含远程仓库的认证信息

重要发现:

凭据提取:

  • 用户: cody
  • 密码: jh1usoih2bkjaspwe92

SSH横向移动

使用原因: 自动化SSH密码认证，避免手动输入密码

参数解释:

  • -p: 指定密码参数
  • 使用原因: 测试从Git配置中发现的凭据是否可用于SSH登录

结果: 成功登录，原来这是svc用户的SSH密码！

6. 权限提升 (Privilege Escalation)

Sudo权限检查

使用原因: 检查当前用户可以以root权限执行哪些命令

发现sudo权限:

脚本目录分析

发现: 多个脚本文件，权限为 --x--x--x（只有执行权限，无读权限）

脚本功能测试

使用原因: 测试脚本的正常功能，了解其工作原理为漏洞利用做准备

输出分析: 脚本显示Docker容器、端口映射、Apache状态等信息

相对路径漏洞分析

通过测试发现：脚本接受参数 full-checkup，很可能会在当前目录查找并执行 full-checkup.sh 文件

漏洞利用

使用原因: 在/tmp目录创建恶意脚本，该目录通常有写权限

脚本解释:

  • 将svc用户添加到sudoers文件
  • NOPASSWD: ALL: 允许无密码执行任何sudo命令
  • 使用原因: 通过修改sudoers文件获得持久的root权限

使用原因: 确保脚本有执行权限

使用原因: 利用sudo权限执行system-checkup.py，触发我们的恶意脚本

获取Root权限

使用原因: 利用修改后的sudoers配置切换到root用户

结果: e8362e463fa2ad8fdeff8a45ef8983ef

关键漏洞总结

  1. Searchor 2.4.0 命令注入 - 初始访问点
  2. Git凭据泄露 - 横向移动到稳定SSH
  3. Sudo权限滥用 + 相对路径漏洞 - 权限提升到root

攻击链: Web漏洞 → 反向shell → 信息收集 → SSH访问 → 权限提升 → 完全控制
